工厂型SKILL是用来创建自定义SKILL的SKILL，用户可以根据工厂SKILL的引导，创建自定义的SKILL。
```
用户触发
├─ 创建指令（/create-ex、帮我创建前任skill等）
│  └─ 主流程：创建新前任Skill
│     ├─ Step1 基础信息录入
│     │  ├─ 花名/代号
│     │  ├─ 基本信息
│     │  └─ 性格画像
│     ├─ Step2 原材料导入
│     │  ├─ A 微信聊天记录 → Bash + wechat_parser.py
│     │  ├─ B QQ聊天记录 → Bash + qq_parser.py
│     │  ├─ C 社交媒体内容 → Read + Bash + social_parser.py
│     │  ├─ D 照片 → Bash + photo_analyzer.py
│     │  └─ E 直接粘贴/口述 → 文本录入
│     ├─ Step3 原材料分析
│     │  ├─ 线路A：Relationship Memory分析
│     │  └─ 线路B：Persona分析
│     ├─ Step4 生成预览并确认
│     └─ Step5 文件写入
│        ├─ Bash创建目录结构
│        ├─ Write写入memory.md
│        ├─ Write写入persona.md
│        ├─ Write写入meta.json
│        └─ Write写入SKILL.md
├─ 进化指令（/update-ex、追加、不对、ta不会这样说等）
│  ├─ 模式1：追加记忆/内容
│  │  ├─ 读取新原材料并解析
│  │  ├─ Read读取现有memory.md、persona.md
│  │  ├─ 增量内容合并分析
│  │  ├─ Bash + version_manager.py 版本备份
│  │  ├─ Edit更新对应文件
│  │  ├─ 重生成SKILL.md
│  │  └─ 更新meta.json
│  └─ 模式2：对话纠正
│     ├─ 识别纠正类型（记忆/性格）
│     ├─ 生成Correction记录
│     ├─ Edit追加至对应文件
│     └─ 重生成SKILL.md
├─ 管理指令
│  ├─ /list-exes → Bash + skill_writer.py list
│  ├─ /ex-rollback → Bash + version_manager.py rollback
│  └─ /delete-ex、/let-go → Bash删除目录
└─ 调用指令
   ├─ /{slug} → 完整对话模式
   ├─ /{slug}-memory → 回忆模式
   └─ /{slug}-persona → 性格模式
```
